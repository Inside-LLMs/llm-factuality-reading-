# DoLa: Decoding by Contrasting Layers Improves Factuality in LLMs (ICLR 2024)
*ICLR 2024*

## 1. 论文要解决什么问题？

### 问题

LLM 经常 hallucination（生成事实错误内容）。

例：
Q: Washington State 的首都？
LLM: Seattle ❌
GT: Olympia ✅

目标：

> 不做微调（fine-tuning）
> 不接外部知识库（RAG）
> 只改 decoding，就提升 factuality

## 2. 作者为什么认为 hallucination 产生？

作者观察：Transformer 不同层存储的信息不同：

- 早层（early layers）：syntax / surface pattern / language fluency
- 高层（later layers）：semantics / factual knowledge

事实 token 的层级行为：

- 错误但“合理”的 token（Seattle）：所有层概率都高
- 正确 token（Olympia）：随着层数增加概率逐渐升高

因而结论是：事实知识在后层逐渐注入。

核心假设：

> 对比高层与低层输出，能放大 factual knowledge。

## 3. DoLa 的核心方法

DoLa = Decoding by Contrasting Layers。

普通解码只用最终层：

```text
p(x_t|x_<t) = softmax(φ(h_t^(N)))
```

其中：

- h_t^(N)：final layer hidden state
- φ(·)：LM head

DoLa 增加 early exit：

```text
q_j(x_t|x_<t) = softmax(φ(h_t^(j)))
```

定义：

- mature layer：q_N
- premature layer：q_M

对比公式：

```text
F(q_N, q_M) = log (q_N(x_t))/(q_M(x_t))
```

最终预测：

```text
hat p(x_t|x_<t) = softmax(log(q_N / q_M))
```

直觉是：如果某个 token 在 final layer 概率高而早期层概率低，
则该 token 的对比得分会被放大。

## 4. 如何选 premature layer？

作者没有固定 M，而是动态选择。

### Dynamic Premature Layer Selection

用 Jensen-Shannon Divergence：

```text
d(q_N, q_j) = JSD(q_N || q_j)
```

选择差异最大的层：

```text
M = argmax_j in J JSD(q_N || q_j)
```

意义是：与 final layer 差异越大，后续层注入的 factual knowledge 越明显。

## 5. Contrast 的实际计算

DoLa 采用类似 Contrastive Decoding 的形式：

```text
F(q_N, q_M) = log (q_N(x_t))/(q_M(x_t))
```

最终：

```text
hat p = softmax(log(q_N / q_M))
```

这一设计让“后层新增事实信号”得到强化。

## 6. Adaptive Plausibility Constraint (APC)

为了避免把低质量 token 也放大，DoLa 加入 APC。

只保留满足：

```text
V_head = x_t : q_N(x_t) ≥ α max_w q_N(w) 
```

否则令：

```text
F = -infty
```

论文取：

```text
α = 0.1
```

作用是避免 false positive / false negative。

## 7. 完整推理流程

输入：Prompt
↓
获取每层 logits
↓
计算若干层的 q_j
↓
计算 JSD(q_N, q_j)
↓
选出 M = argmax JSD
↓
计算对比得分：log(q_N / q_M)
↓
softmax
↓
生成 token

## 8. 关键实验结果

### TruthfulQA

LLaMA-7B：
- 原始 MC2 = 40.6
- DoLa MC2 = 63.8
- 提升 +23.2 points

### Open-ended generation

Truthfulness 提升 12–17%。
且无需：
- RAG
- finetune
- extra training

### CoT 推理

- StrategyQA 提升约 1–4%
- GSM8K 提升约 2%

说明 DoLa 不只增强 factuality，也对 reasoning 有帮助。

### 延迟成本

额外 latency 仅 1.01× 1.08×，几乎可忽略。

## 9. 与 Contrastive Decoding (CD) 的区别

CD：大模型 - 小模型

```text
Expert - Amateur
```

DoLa：同一个模型内部的层对比

```text
LateLayer - EarlyLayer
```

优点：无需第二个模型、额外训练或更多显存。

## 10. 本文最大贡献

一句话：

> 作者发现 factual knowledge 在 Transformer 后层逐渐形成，
> 因此通过“最终层 - 早期层”的 decoding，
> 放大真实知识，从而减少 hallucination。

核心贡献：

1. layer contrast for decoding
2. dynamic layer selection via JSD
3. zero-training factuality improvement

## 11. 局限

作者承认：

1. 只能利用模型已有信息，预训练的错误知识无法纠正
2. 不接外部知识库，无法替代 RAG
3. 主要关注 factuality，对 alignment/RLHF 仍未验证

## 12. Take-away

- 问题：LLM hallucination
- 观察：事实知识在后层形成
- 方法：DoLa = LateLayer - EarlyLayer
- 选层：argmax JSD
- 解码：softmax(log(q_N / q_M))
- 效果：显著提升 factuality
- 优点：无需 fine-tune / RAG / extra model
